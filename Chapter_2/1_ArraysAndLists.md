# Массивы и списки

Массивы и списки — это структуры данных для хранения набора элементов (чисел, строк, объектов и т. д.).

## Массивы

- **Что такое массив**: непрерывный блок памяти, в котором все элементы одного типа расположены подряд.
- **Доступ по индексу**: к каждому элементу можно обратиться по его номеру (индексу) за \(O(1)\) — очень быстро.
- **Размер**: как правило, фиксированный; чтобы добавить много новых элементов, часто приходится создавать новый массив большего размера и копировать данные.

Пример на PHP:

```php
$numbers = [10, 20, 30, 40];

// доступ по индексу
echo $numbers[0]; // 10
echo $numbers[3]; // 40

// изменение элемента
$numbers[1] = 200; // [10, 200, 30, 40]

// добавление в конец массива
$numbers[] = 50; // [10, 200, 30, 40, 50]
```

## Списки

- **Что такое список**: в теории это структура, где элементы связаны друг с другом (часто через указатели).
- **Добавление и удаление**: вставка или удаление элемента в начале или середине списка может занимать \(O(1)\), если у вас уже есть ссылка на нужное место.
- **Доступ по индексу**: чтобы добраться до элемента с индексом *i*, нужно пройти по списку от начала, поэтому это \(O(n)\).

В PHP нет «чистого» связного списка из коробки, но его можно смоделировать с помощью классов:

```php
<?php

class Node
{
    public int $value;
    public ?Node $next;

    public function __construct(int $value, ?Node $next = null)
    {
        $this->value = $value;
        $this->next  = $next;
    }
}

// создание списка: 10 -> 20 -> 30 -> null
$node3 = new Node(30, null);
$node2 = new Node(20, $node3);
$head  = new Node(10, $node2);

// проход по списку
$current = $head;
while ($current !== null) {
    echo $current->value . PHP_EOL;
    $current = $current->next;
}
```

## Когда что использовать

- **Массив**:
  - нужен быстрый произвольный доступ к элементам по индексу;
  - количество элементов заранее примерно известно и меняется не очень часто.

- **Список**:
  - часто добавляете/удаляете элементы в середине структуры;
  - размер сильно и непредсказуемо меняется;
  - произвольный доступ по индексу не критичен.

Понимание различий между массивами и списками важно для выбора правильной структуры данных и эффективных алгоритмов.
